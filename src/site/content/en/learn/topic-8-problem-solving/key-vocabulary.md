## How to use this module

For each phrase or term:
1. Say it out loud 3 times.
2. Use it in a sentence about your own work.
3. Practice using it naturally in an answer to a practice question.

**Goal:** Zero hesitation when these words come up in an interview.

## Daily drill

Pick 3–5 phrases from this list each day. Use each one out loud in a complete sentence before moving on.

---

## Thinking & Reasoning

| Phrase | When to Use |
|--------|-------------|
| "Let me think through this step by step..." | Opening a problem-solving answer |
| "The first thing I'd check is..." | Starting your diagnostic process |
| "My hypothesis is that the issue is..." | Stating your theory before testing |
| "Let me verify that assumption..." | Showing rigor in your thinking |
| "Based on what I'm seeing, I believe..." | Connecting evidence to conclusion |

## Debugging & Analysis

| Term / Phrase | Usage Example |
|---------------|---------------|
| bottleneck | "The database query was the bottleneck — it accounted for eighty percent of response time." |
| profiling | "I ran profiling on the API and found three queries taking over five hundred milliseconds each." |
| benchmark | "We benchmark every release against the previous version's performance numbers." |
| regression | "The deploy introduced a performance regression — p95 latency doubled." |
| root cause | "The root cause turned out to be a missing index on the new status column." |
| correlation ID | "We trace every request with a correlation ID so we can follow it across services." |

## Trade-offs & Decisions

| Phrase | When to Use |
|--------|-------------|
| "The trade-off between X and Y is..." | Explaining competing options |
| "I'd measure performance by..." | Defining success criteria |
| "The root cause turned out to be..." | Concluding your investigation |
| "To optimize this, I would..." | Proposing your solution |
| "I'd choose X over Y because the primary constraint is..." | Defending a decision |
| "The deciding factor was..." | Explaining what tipped the balance |

## Code Quality

| Term / Phrase | Usage Example |
|---------------|---------------|
| tech debt | "We track tech debt on the board and allocate twenty percent of each sprint to pay it down." |
| refactoring | "I refactored the payment module to separate concerns before adding the new feature." |
| clean code | "Clean code means the next engineer can read and modify it without asking me." |
| code smell | "Duplicated logic across three services was a code smell that led us to extract a shared library." |
