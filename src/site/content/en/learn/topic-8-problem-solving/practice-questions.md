## How to use this module

For each question:
1. **First pass:** Answer out loud with notes allowed.
2. **Second pass:** Answer out loud — no notes, no preparation.
3. **Record yourself** on at least one question per session and listen back.

**Target:** Each answer should feel automatic — natural, structured, confident.

## Scoring yourself

After each answer, ask:
- Did I think out loud clearly?
- Did I follow a structured approach (not random guessing)?
- Did I give a real example from my experience?
- Did the answer take 2–4 minutes?

## Question 1 — How would you debug a production issue where response times suddenly increased?

**What the interviewer expects:** Structured approach, not panic. Check metrics first, identify the component, form a hypothesis, test it. Show you've done this before.

**Structure your answer:** Initial triage (dashboards, logs, recent deploys) → Isolate the component → Form hypothesis → Test → Fix → Post-mortem.

## Question 2 — Walk me through how you approach a problem you've never seen before

**What the interviewer expects:** Comfort with ambiguity. Research, decompose, prototype, iterate. Show you don't freeze when facing the unknown.

**Structure your answer:** Acknowledge the unknown → Break it into smaller problems → Research and prototype → Test assumptions → Iterate → Real example.

## Question 3 — How do you decide when to optimize code vs ship it as-is?

**What the interviewer expects:** Pragmatism. Not everything needs to be optimized. Decision criteria: user impact, scale, deadline, maintenance cost. Show engineering judgment.

**Structure your answer:** Decision criteria (impact, scale, deadline) → When to ship as-is → When to optimize first → How you communicate the trade-off → Real example.

## Question 4 — What is tech debt and how do you manage it in a fast-moving team?

**What the interviewer expects:** Nuanced view. Not all tech debt is bad. Track it, prioritize it, allocate time to pay it down. Show you balance speed with quality.

**Structure your answer:** Define tech debt → Why it accumulates → How you track it → How you prioritize which to fix → How you allocate time → Real example.

## Question 5 — How do you handle disagreement about a technical approach with a teammate?

**What the interviewer expects:** Maturity. Listen, understand their reasoning, present your data, find common ground or escalate constructively. No ego.

**Structure your answer:** Listen first → Understand their reasoning → Present your evidence → Find common criteria → If still stuck, propose a test or escalate → Real example.
