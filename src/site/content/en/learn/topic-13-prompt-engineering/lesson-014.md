## Session goal

Deliver a 5-minute overview of your LLM Best Practices Guide as if presenting it to your team for the first time.

## Shadowing passage — opening for the team presentation

> I want to share a document we've been building over the last few months: our LLM Best Practices Guide. The goal of this document is simple — it captures everything we've learned about using LLMs in production, so that the next engineer on the team doesn't have to rediscover it the hard way. The guide covers six areas. First, prompt templates: the standard structure we use — system prompt, user prompt, assistant prefill — and the common patterns: zero-shot, few-shot, chain-of-thought, and structured output. Second, reliability: how we handle retries, fallbacks, and context window limits. Third, cost control: our model selection matrix — which model tier for which task type — and our approach to prompt caching and token budgeting. Fourth, security: PII redaction, prompt injection defenses, and our testing checklist for new prompts. Fifth, evaluation: how we build eval sets, run regression tests, and use LangSmith for prompt versioning. Sixth, known gotchas: the specific issues we've hit in production — over-refusal patterns, brittle prompts, silent quality degradation — and how we resolved each one. The document is a living document: it should be updated when we discover something new or when we fix something that broke. I'm asking each of you to read section one and two this week and tell me if there's anything we've missed.
