## Session goal

Explain a sprint to a product manager who doesn't code.

## Shadowing passage

> When I communicate engineering progress to a product stakeholder, my rule is: frame everything in terms of user outcomes and time, not technical tasks. Not 'implemented database query optimization for the analytics pipeline' — that means nothing to a non-engineer. Instead: 'users now see their quarterly report generate in twelve seconds instead of ninety — the analysis phase is fast enough to run on demand.' That translation takes extra effort, but it's what makes non-engineering stakeholders feel informed and able to make decisions. My sprint communication format has three components. Work done: what shipped, framed as user-facing changes. Blockers and decisions: if a technical decision depends on a product choice, I state that explicitly — 'I need to know whether we want to support Instagram integration before I complete the platform layer, because it changes the data model.' Next milestone: one week out, stated as a user outcome — 'by next Friday, customers will be able to preview and edit generated briefs before approval.' I never present technical complexity as a reason for delay without an alternative. If something is taking longer than planned, I explain the actual constraint, offer options, and make a specific recommendation. Options plus recommendation is the format — not 'this is complicated, it will take longer.'

## 30-minute protocol

**Minutes 1–5** — Read the passage silently twice. Focus on meaning, not pronunciation.

**Minutes 6–15** — Shadow aloud with the text visible. 4 full passes. Match rhythm and stress.

**Minutes 16–22** — Shadow without the text. 3 passes. Accept mistakes — keep moving.

**Minutes 23–28** — Record yourself once. Play it back immediately. Note one phrase to improve.

**Minutes 29–30** — Write the phrase you struggled with. Say it 10 times slowly, then at full speed.

## Anti-Translation Drill — Interview Mode *(5 min)*

Someone just asked you the Reflection question in a **real interview**. Answer now.

- You have **60 seconds**. No pauses longer than 3 seconds.
- Fill gaps with: *"Let me think about that…"* / *"The way I see it…"* / *"To give you a concrete example…"*
- **Record this answer** separately from your shadowing recording.
- Play it back. Did you sound like someone they'd hire?

> Goal: Simulate interview pressure. The ability to speak under pressure without translating is what separates "good English" from "interview-ready English".
