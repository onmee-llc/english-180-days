## Part 1 — Skim (5 minutes)

The model specification is structured as a policy document, not a blog post. Look for:
- Section titles that indicate scope (e.g., "operators and users", "avoiding harm")
- Any numbered lists of principles or priorities
- The language of conditions ("when…", "unless…", "in cases where…")

After skimming, answer:
> "What are Claude's stated priorities when its values come into conflict with user requests?"

**Starter vocabulary:**

| Term | Definition |
|------|-----------|
| principal hierarchy | the ordered set of entities (Anthropic, operator, user) whose instructions the model should follow |
| operator | the company or individual who accesses Claude via API and deploys it in a product |
| system prompt | instructions provided at the start of a conversation that configure the model's behavior |
| default behavior | how the model behaves when no special instructions are provided |
| non-default behavior | behavior that must be explicitly unlocked by an operator or user |
| latitude | the degree of freedom the model has to use its own judgment |
| harm avoidance | the model's process of assessing and minimizing potential harms in its outputs |
